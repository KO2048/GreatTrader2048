# 大作手 · The Great Trader
## 交易系统 PRD v3.1

> **文档定位**：交易引擎的唯一权威定义。
> 只描述交易行为本身——开仓、持仓、止盈、止损、强平、手动平仓、手续费。
> 关卡目标、本金 Tier 升级、叙事进程不属于本文档，归属《关卡系统 PRD》。
> Combo、卡牌效果、动画、音效不属于本文档，归属《卡牌系统 PRD》。

## 一、设计原则

### 1.1 交易引擎边界

```
引擎负责且只负责：
  开仓      → 建立仓位，扣保证金 + 开仓手续费
  持仓      → 每 tick 实时计算浮盈亏
  止盈触发  → 达到 tpRate，自动平仓，emit 'pmax'
  止损触发  → 达到 slRate，自动平仓，emit 'sl'
  强平触发  → 亏损 100%，强制平仓，emit 'liq'
  手动平仓  → 玩家主动操作，emit 'manual'
  手续费    → 开仓和平仓各收一次

引擎不负责：
  Combo 计数与结算    → 卡牌系统监听 emit 事件后处理
  动画 / 音效         → UI 层监听 emit 事件后处理
  卡牌触发与修改      → 卡牌系统负责
  关卡目标判定        → 关卡系统负责
  本金 Tier 管理      → 关卡系统负责
```

### 1.2 核心数学公式

毛盈亏（Gross PnL）：

  GrossPnL = margin × lev × (ΔP / P_entry) × dir

净盈亏（Net PnL，含手续费）：

  NetPnL = margin × [ lev × (ΔP / P_entry) × dir − 2 × feeRate ]

止盈触发时净盈亏（ΔP/P_entry × lev = tpRate）：

  NetPnL_tp = margin × (tpRate − 2 × feeRate)

盈亏平衡所需最小价格移动：

  break_even_move = 2 × feeRate / lev

含手续费保本胜率（近似）：

  W_min ≈ slRate / (tpRate + slRate)

其中：
  dir      = +1（做多）或 −1（做空）
  feeRate  = 单边手续费率，初始值 0.10
  开仓费 + 平仓费 = margin × feeRate × 2

## 二、杠杆体系

### 2.1 可选杠杆档位

| 档位 | 杠杆 | 止盈触发价格移动（tpRate=100%） | 定位 |
|------|------|-------------------------------|------|
| 1 | 1x | 100% | 教育 / 极慢 |
| 2 | 5x | 20% | 慢 |
| 3 | 10x | 10% | 中 |
| 4 | 50x | 2% | 较快 |
| 5 | 100x | 1% | 快 |
| 6 | 200x | 0.5% | 快 |
| 7 | 500x | 0.2% | 极快 |
| 8 | 1000x | 0.1% | 老虎机感 |
| 9 | 2000x | 0.05% | 老虎机感 |
| 10 | 5000x | 0.02% | 推币机感 ★核心体验 |
| 11 | 10000x | 0.01% | 推币机极限 |

> 核心体验杠杆：5000x ～ 10000x。
> 低倍杠杆（1x ～ 50x）作为策略选项保留，不是主要体验路径。

### 2.2 杠杆的核心作用

```
杠杆只决定触发速度，不决定单笔收益金额。

NetPnL_tp = margin × (tpRate − 2 × feeRate)  ← 与杠杆无关

杠杆越高 → 触发所需价格移动越小 → 单位时间触发次数更多
杠杆越低 → 需要大幅价格移动才能触发 → 反馈慢
```

### 2.3 开仓面板布局

多空双区同时显示，不使用 Tab 切换。
「触发移动%」= tpRate / lev，实时反映当前 tpRate。

```

│ ▌ 做多 [Long]                    [绿色顶边框]    │
│  杠杆     触发移动%    $本金1  $本金2  $本金3    │
│  1000x     0.1%        [按]    [按]    [按]      │
│  2000x     0.05%       [按]    [按]    [按]      │

├──────────────────────────────────────────────────┤
```

## 三、止盈率 / 止损率

### 3.1 止盈率（tpRate）

含义：持仓盈利率达到此阈值，系统自动平仓锁定利润。

| 属性 | 值 |
|------|----|
| 初始默认值 | 100% |
| 最低值 | 10% |
| 最高值 | 10000%（卡牌可扩展） |
| 唯一修改入口 | 卡牌 |

```
触发价格移动 = tpRate / lev

5000x 杠杆下：
  tpRate=100%   → 移动 0.02%（几乎必触发）
  tpRate=1000%  → 移动 0.2%
  tpRate=5000%  → 移动 1%（需判断方向）
  tpRate=10000% → 移动 2%（高难度，单笔巨大）
```

> 止盈率 ≠ 保证收益。高 tpRate 打开收益上限，但不保证触发。
> 玩家可等待自动触发，或在合适时机手动平仓——两种退出模式并存。

### 3.2 止损率（slRate）

含义：持仓亏损率达到此阈值（且 <100%），提前止损出场，实际亏损小于保证金全额。

| 属性 | 值 |
|------|----|
| 初始默认值 | 100%（= 无止损，等待强平） |
| 最低值 | 10% |
| 最高值 | 100% |
| 唯一修改入口 | 卡牌 |

> slRate = 100% 时，止损永不提前触发，行为与 v1.0 完全一致。

### 3.3 不对称 TP/SL 的策略意义

```
不对称设置（tpRate >> slRate）→ R:R > 1
  例：tpRate=200%, slRate=50% → R:R = 4:1
  → 胜率仅需 25% 即可长期盈利

多空不对称对冲：
  多仓高 tpRate + 空仓低 tpRate
  → 一个方向的强平 / 止损 = 另一个方向止盈的确认信号
  → 两个方向互为对冲，捕捉双向波动
```

### 3.4 参数边界

| 参数 | 最小值 | 最大值 |
|------|--------|--------|
| tpRate | 10% | 10000%+ |
| slRate | 10% | 100% |

## 四、手续费体系

### 4.1 基准定义

计费基准：本金（margin），非名义仓位（margin × lev）

| 操作 | 费率 | 公式 |
|------|------|------|
| 开仓 | 本金 × 10% | margin × feeRate |
| 平仓 | 本金 × 10% | margin × feeRate |
| 单笔合计 | 本金 × 20% | margin × feeRate × 2 |

> feeRate 初始值 = 0.10，唯一修改入口：卡牌，下限 0（全免）。

### 4.2 手续费的策略调节作用

```
高频策略（多次开平仓）→ 手续费磨损最重 → 需 fee 减免卡才能正 EV
低频策略（少次大仓位）→ 手续费影响最小 → 天然低摩擦
对冲策略（多仓并行）  → 开仓次数多      → 手续费为隐性成本压制

feeRate 是系统最优雅的宏观调节阀：
  feeRate 越高 → 抑制高频策略优势
  卡牌降低 feeRate → 解锁高频策略正 EV 空间
```

### 4.3 存疑备忘

> 按本金收费导致低杠杆手续费占比偏重，高杠杆摩擦成本相对更低。
> 是否需要额外机制制衡极高杠杆使用倾向，待后续版本评估。

## 五、三大触发与结算

### 5.1 触发条件

```
pnlRate = GrossPnL / margin  // 正=盈利，负=亏损，单位 %

[1] pnlRate >= pos.tpRate               → trigPmax()   止盈
[2] pnlRate <= -pos.slRate
    且 pos.slRate < 100                  → trigSL()     止损
[3] pnlRate <= -100                     → liq()        强平
[4] 玩家点击平仓                        → closePos()   手动平仓
```

### 5.2 四种结算对比

| | 止盈 trigPmax | 止损 trigSL | 强平 liq | 手动 closePos |
|--|-------------|------------|---------|--------------|
| **触发** | pnlRate ≥ tpRate | pnlRate ≤ −slRate | pnlRate ≤ −100% | 玩家操作 |
| **余额** | +margin + NetPnL | −margin × slRate% | −margin（全损） | ±当前 NetPnL |
| **emit** | `'pmax'` | `'sl'` | `'liq'` | `'manual'` |

> 所有后续处理（Combo、动画、卡牌触发）由外部系统监听 emit 事件处理，引擎不感知。

### 5.3 完整状态机

```
openPos()
  │
  ▼
status = 'open'
  │
  ├─ pnlRate >= tpRate ──────────→ trigPmax()   emit('pmax')
  │                                余额 += margin + NetPnL
  │
  ├─ pnlRate <= -slRate
  │  (slRate < 100) ─────────────→ trigSL()     emit('sl')
  │                                余额 -= margin × slRate%
  │
  ├─ pnlRate <= -100% ───────────→ liq()        emit('liq')
  │                                余额 -= margin
  │
  └─ 玩家手动 ───────────────────→ closePos()   emit('manual')
                                   余额 ±= 当前 NetPnL
```

## 六、仓位管理

### 6.1 仓位数据结构

```javascript
pos = {
  id:      唯一标识（时间戳）,
  dir:     'long' | 'short',
  lev:     杠杆倍数,
  margin:  保证金金额,
  entry:   开仓价格,
  tpRate:  止盈率（默认 G.globalTP，卡牌可修改）,
  slRate:  止损率（默认 G.globalSL，卡牌可修改）,
  status:  'open' | 'pmax' | 'sl' | 'liq' | 'manual',
  ts:      开仓时间戳
}

// 全局交易参数（唯一修改入口：卡牌）
G.globalTP  = 100    // 止盈率初始值（%）
G.globalSL  = 100    // 止损率初始值（%）
G.feeRate   = 0.10   // 单边手续费率
```

### 6.2 仓位上限

| 维度 | 限制 |
|------|------|
| 单资产最大仓位数 | 6 个 |
| 账户总仓位上限 | 20 个 |

### 6.3 分仓隔离原则

```
每笔仓位最大亏损 = 该仓保证金（margin）
一仓强平不影响其他仓位
账户余额永远 >= 0，不会穿仓至负数
```

## 七、策略多样性与平衡

### 7.1 无通吃策略的数学保证

| 想要… | 必须牺牲… |
|-------|---------|
| 高胜率（低 tpRate） | 单笔收益小 |
| 高单笔收益（高 tpRate） | 胜率降低 |
| 触发更快（高杠杆） | 强平也更快 |
| 低手续费（fee 卡） | 占用卡牌槽位 |
| 大本金（margin 升级卡） | 占用卡牌槽位 |

### 7.2 主要策略路线及内生代价

| 策略路线 | 天然优势 | 结构性代价 |
|---------|---------|----------|
| 高频止盈流（低 tpRate） | 触发频繁 | 手续费磨损重 |
| 低频厚利流（高 tpRate） | 单笔巨大 | 胜率低，等待长 |
| 对冲套利流（多空并行） | 双向捕捉波动 | 止损必然触发，手续费双倍 |
| 逆势博弈流 | 精准时机巨盈 | 强平风险极高 |
| 稳定复利流（均衡 R:R） | EV 稳定 | 无单项极值 |

## 八、核心函数索引

| 函数 | 功能 | emit |
|------|------|------|
| `openPos(lev, margin, dir)` | 开仓，扣保证金 + 开仓费 | — |
| `closePos(id)` | 手动平仓，结算 NetPnL | `'manual'` |
| `checkPosition(pos)` | 每 tick 检查三大触发条件 | — |
| `trigPmax(pos)` | 止盈结算 | `'pmax'` |
| `trigSL(pos)` | 止损结算 | `'sl'` |
| `liq(pos)` | 强平结算 | `'liq'` |
| `revPos(id)` | 反手：平仓 + 反向开仓 | — |

## 九、向后兼容性

默认值（tpRate=100, slRate=100, feeRate=0.10）无卡牌介入时：
- 止盈行为与 v1.0 Pmax 完全一致
- 无止损（slRate=100 → trigSL 永不触发）
- 手续费为 v3.x 新增（v1.0 无手续费）
