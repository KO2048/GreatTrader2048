## 一、MVP 核心理念

### 1.1 一句话定位
> 极简交易核心循环：开仓 → 止盈/止损/强平 → 余额变化 → 选择卡牌强化参数 → 达成关卡目标。

### 1.2 设计优先原则
- **试错友好**：L1-1 余额目标宽松，失败成本低，让玩家在反复尝试中理解止盈、止损、杠杆与保证金的关系。
- **机制透明**：所有参数变化（止盈率、止损率、保证金等）均由卡牌直接反映，无隐藏公式。
- **乘法爽感**：借鉴 Balatro，提供加算（+1倍收益）与乘算（×2倍收益）两类成长路径，乘法卡为构筑核心。

## 二、核心数值参数

### 2.1 交易基础参数

| 参数 | 初始值 | 说明 |
| :--- | :--- | :--- |
| **初始余额** | $10 | 第一关起始资金 |
| **默认止盈率** | 100% | 盈利达到保证金 100% 时自动平仓 |
| **默认止损率** | 80% | 亏损达到保证金 80% 时自动平仓（保护本金） |
| **手续费率** | 15% | 开仓、平仓各收取保证金 × 15%，单边独立计算 |
| **初始最大保证金** | $1 | 每笔交易保证金上限（通过卡牌提升） |
| **单方向最大持仓数** | 6 个 | 多单、空单分别计算，可同时双边持仓 |
| **可选杠杆档位** | 100x, 200x, 500x, 1000x, 2000x, 5000x | 玩家手动选择 |

### 2.2 关卡目标与失败线

| 关卡 | 初始余额 | 目标余额 | 失败条件（余额 ≤） | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **L1-1** | $10 | $20 | $0 | 试错关，余额翻倍即可通关，失败代价低 |
| **L1-2** | 继承上关 | $50 | $10 | 需赚取 $30，开始有回撤压力 |
| **L1-3** | 继承上关 | $100 | $30 | 目标翻倍，失败线抬升，考验稳定性 |
| **L1-4** | 继承上关 | $300 | $70 | 第一章毕业关，需累计盈利 $290 |

> **失败处理**：余额低于或等于失败线时，本关立即结束并提示失败，需从本章第一关重新开始。

## 三、交易核心公式

### 3.1 盈亏计算

**止盈触发条件**：`浮动盈亏 / 保证金 ≥ 止盈率`

**止损触发条件**：`浮动盈亏 / 保证金 ≤ -止损率`

**强平触发条件**：`浮动盈亏 / 保证金 ≤ -100%`（亏损全部保证金）

**净盈亏公式（以多头为例）**：
```
毛盈亏 = 保证金 × 杠杆 × (平仓价 - 开仓价) / 开仓价
手续费 = 保证金 × 15% × 2   （开仓 + 平仓）
净盈亏 = 毛盈亏 - 手续费
```
*空头方向价格变化取相反数。*

**简化理解**（不考虑价格波动细节）：
- 止盈时净收益 ≈ `保证金 × (止盈率 - 30%)`（因双边手续费合计 30%）
- 止损时净亏损 ≈ `保证金 × (止损率 + 30%)`

### 3.2 盈亏平衡所需最小价格移动
```
价格变动需求 = (2 × 15%) / 杠杆 = 30% / 杠杆
```
- 100x 杠杆：需 0.3% 价格移动
- 5000x 杠杆：需 0.006% 价格移动

## 四、卡牌系统（MVP 基础类型）

所有卡牌效果均直接修改上述核心参数。卡牌分为**数值加算**与**倍率乘算**两类，乘算卡更为稀有。

### 4.1 保证金调整卡

| 卡牌类型 | 效果示例 | 稀有度倾向 |
| :--- | :--- | :--- |
| **保证金增加卡** | 最大保证金 +1 U / +5 U / +10 U | 普通～稀有 |
| **保证金乘法卡** | 最大保证金 ×2 / ×3 / ×5 | 稀有～传说 |

### 4.2 止盈率调整卡

| 卡牌类型 | 效果示例 | 说明 |
| :--- | :--- | :--- |
| **止盈率增加卡** | 止盈率 +5% / +10% / +20% | 提高单笔盈利上限，但触发难度略增 |
| **止盈率减少卡** | 止盈率 -5% / -10% / -20% | 降低触发门槛，适合高频薄利流派 |

### 4.3 止损率调整卡

| 卡牌类型 | 效果示例 | 说明 |
| :--- | :--- | :--- |
| **止损率增加卡** | 止损率 +5% / +10% / +20% | 放宽止损线，给行情更多回调空间 |
| **止损率减少卡** | 止损率 -5% / -10% / -20% | 收紧止损，保护本金，但容易提前离场 |

### 4.4 手续费调整卡

| 卡牌类型 | 效果示例 | 说明 |
| :--- | :--- | :--- |
| **手续费减少卡** | 手续费率 -1% / -3% / -5%（单边） | 降低摩擦成本，高频构筑核心 |

### 4.5 杠杆收益强化卡（借鉴 Balatro 加算/乘算）

| 卡牌类型 | 效果示例 | 结算方式 | 稀有度 |
| :--- | :--- | :--- | :--- |
| **杠杆收益增加卡** | 某方向某杠杆收益 **+1 倍** | 最终收益 = 原净盈亏 × (1 + 1) | 普通～稀有 |
| **杠杆收益乘法卡** | 某方向某杠杆收益 **×2 倍** | 最终收益 = 原净盈亏 × 2 | 稀有～传说 |

> **说明**：
> - “+1 倍”指在原有收益基础上额外增加 1 倍本金对应的利润（即翻倍效果）。
> - “×2 倍”是直接对最终净盈亏进行乘法运算。
> - 多张同向收益卡叠加时，**先计算所有加算，再计算乘算**（Balatro 式叠加顺序）。

## 五、发牌与商店（MVP 简化规则）

- **通关奖励**：每关通关后出现商店，随机展示 3 张候选卡牌，玩家选择 1 张永久生效。
- **随机池**：从上述卡牌类型中完全随机抽取，**不引入信号标签权重系统**（v2.0 实现）。
- **保底**：若玩家连续 3 次商店未遇到任何保证金提升类卡牌，下一次商店强制将一张候选替换为随机保证金增加卡。

## 六、行情数据（MVP 简化方案）

- **数据来源**：使用预设的伪随机价格序列，或接入静态历史 K 线数据（如 BTC 2024 年某段 1 分钟线）。
- **波动特征**：确保存在趋势、震荡、插针等多种形态，但无需实现复杂的阶段参数控制。
- **推进方式**：手动点击“下一根 K 线”或自动按固定时间间隔（如 1 秒）推进。
